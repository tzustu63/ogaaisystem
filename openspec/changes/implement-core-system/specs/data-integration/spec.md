## ADDED Requirements

### Requirement: 檔案匯入功能（Level 1）
系統 SHALL 支援 CSV/Excel 檔案匯入，包括欄位映射、資料驗證、匯入預覽、版本控管。

#### Scenario: 上傳檔案
- **WHEN** 使用者上傳 CSV/Excel 檔案
- **THEN** 系統檢查檔案格式與大小
- **AND** 系統要求選擇對應的資料表
- **AND** 系統顯示檔案預覽

#### Scenario: 欄位映射
- **WHEN** 使用者進行欄位映射
- **THEN** 系統提供拖拉對應介面
- **AND** 系統顯示來源欄位與目標欄位
- **AND** 系統驗證必填欄位是否已映射

#### Scenario: 資料驗證
- **WHEN** 使用者確認匯入
- **THEN** 系統執行資料驗證（格式/必填/範圍）
- **AND** 系統顯示驗證結果與錯誤提示
- **AND** 系統允許修正錯誤後重新匯入

#### Scenario: 匯入預覽
- **WHEN** 資料驗證通過
- **THEN** 系統顯示匯入預覽（前 N 筆資料）
- **AND** 系統顯示匯入筆數統計
- **AND** 系統要求使用者確認後才執行匯入

#### Scenario: 版本控管
- **WHEN** 匯入完成
- **THEN** 系統記錄匯入資訊：時間、操作人、檔案名稱、筆數
- **AND** 系統支援版本回溯與對比
- **AND** 系統供稽核時查證數據來源

### Requirement: 校內系統對接（Level 2）
系統 SHALL 支援與校內系統對接，優先順序：學籍/SIS系統、財務/主計系統、人資/研習系統。

#### Scenario: API 串接
- **WHEN** 系統對接校內系統 API
- **THEN** 系統支援即時/每日/每週同步頻率
- **AND** 系統處理 API 認證與授權
- **AND** 系統記錄同步狀態與錯誤

#### Scenario: ETL 定時同步
- **WHEN** 校內系統無 API 時使用 ETL
- **THEN** 系統支援定時 ETL 排程
- **AND** 系統支援增量同步與全量同步
- **AND** 系統記錄同步歷程

#### Scenario: 資料同步狀態
- **WHEN** 使用者查看系統對接狀態
- **THEN** 系統顯示各系統的對接狀態
- **AND** 系統顯示最後同步時間
- **AND** 系統顯示同步錯誤（如有）

### Requirement: 外部資料與填報工作台（Level 3）
系統 SHALL 支援 QS/THE 排名填報，包括建立填報指標清單、追蹤填報完整率、資料佐證檢核清單、截止日提醒。

#### Scenario: 建立填報指標清單
- **WHEN** 使用者建立排名填報專案
- **THEN** 系統允許建立填報指標清單
- **AND** 系統允許為每個指標指定資料來源與負責人
- **AND** 系統允許設定截止日期

#### Scenario: 追蹤填報進度
- **WHEN** 使用者查看填報進度
- **THEN** 系統顯示填報完整率
- **AND** 系統標記已完成/進行中/未開始的指標
- **AND** 系統顯示各指標的負責人與狀態

#### Scenario: 資料佐證檢核
- **WHEN** 使用者上傳填報資料
- **THEN** 系統檢查資料佐證是否齊備
- **AND** 系統顯示檢核清單
- **AND** 系統標記缺漏的佐證資料

#### Scenario: 截止日提醒
- **WHEN** 接近填報截止日
- **THEN** 系統自動發送提醒通知
- **AND** 系統顯示剩餘天數
- **AND** 系統在截止日前一天發送緊急提醒
